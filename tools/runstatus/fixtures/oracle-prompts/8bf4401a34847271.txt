> ## Bug: ESC silently discarded while TUI is in `ModeAwaitingLLM`

### What is broken

When a user presses **ESC while an LLM turn is in flight** (spinner active, mode = `ModeAwaitingLLM`), the keypress is silently discarded. The system menu does not open, no transcript feedback is shown, and the in-flight turn is not cancelled. From the user's perspective, the TUI appears completely frozen and unresponsive.

### Root cause

In `internal/tui/tui.go` around line 1251, `routeKey()` guards the ESC‚Üímenu transition to `ModeOnPath | ModeOffPath` only:

```go
if msg.Type == tea.KeyEsc {
    if m.mode == ModeOnPath || m.mode == ModeOffPath {
        m.mode = ModeMenu
        m.menuSystem.Open()
        return m, nil
    }
}
```

When `m.mode == ModeAwaitingLLM`, this guard is skipped entirely. The key then falls through to `m.prompt.Update(msg)`, which passes it to the `charmbracelet/bubbles/textarea` widget ‚Äî a widget that has no ESC binding, so it is silently dropped.

Ctrl+C is handled correctly for `ModeAwaitingLLM` (calls `inFlightCancel`), but ESC is not.

### Reproduction

A deterministic Go test was written and is on disk at:

```
internal/tui/tkt001_esc_repro_test.go
```

The test has two cases:

**`TestTKT001_EscSilentlySwallowedDuringInFlight`** ‚Äî proves the bug. Calls `SimulateSlowHarnessTurnStart` to put the model into `ModeAwaitingLLM` (with a no-op cancel func wired), then sends `tea.KeyEsc`. All three "BUG" assertions pass:
1. Mode remains `ModeAwaitingLLM` ‚Äî menu never opened.
2. Transcript content unchanged ‚Äî no "press Ctrl+C" or similar hint was appended.
3. `HasInFlightCancel` is still `true` ‚Äî `inFlightCancel` was never called; the turn was not cancelled.

**`TestTKT001_ControlEscFromOnPathOpensMenu`** ‚Äî positive control. ESC from `ModeOnPath` correctly transitions to `ModeMenu`. Passes, confirming test helpers are sound.

Run: `cd internal/tui && go test -v -run TestTKT001 .`

Output:
```
--- PASS: TestTKT001_EscSilentlySwallowedDuringInFlight (0.01s)
--- PASS: TestTKT001_ControlEscFromOnPathOpensMenu (0.01s)
```

### Fix direction

Add a `ModeAwaitingLLM` branch to the ESC handler in `routeKey()`. At minimum, show the user a hint ("turn in flight ‚Äî use Ctrl+C to cancel") or call `inFlightCancel` directly and transition mode. The queue-clear branch just above the menu guard is a useful model for the fix pattern.
