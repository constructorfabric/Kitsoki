# Reviewing tests & implementation — produce the testing artifact

You are reviewing the implementation of the fix for **TKT-001** —
*TUI hangs on ESC*.

The proposed fix was:

> ## Bug

When a user presses **ESC while an LLM turn is in flight** (`ModeAwaitingLLM`, spinner active), the keypress is silently discarded. No system menu opens, no transcript hint is shown, and the in-flight turn is not cancelled. The TUI appears completely frozen.

## Root Cause

`routeKey()` in `internal/tui/tui.go:1251` gates the ESC→menu transition on `ModeOnPath | ModeOffPath` only:

```go
if msg.Type == tea.KeyEsc {
    if m.mode == ModeOnPath || m.mode == ModeOffPath {
        m.mode = ModeMenu
        m.menuSystem.Open()
        return m, nil
    }
}
```

When `m.mode == ModeAwaitingLLM` the guard is not entered, the key falls through to `m.prompt.Update(msg)` (the `charmbracelet/bubbles/textarea` widget), which has no ESC binding and silently drops it.

Ctrl+C is handled correctly at line 1187 — it calls `m.inFlightCancel()` and returns — but ESC has no equivalent branch for this mode.

## Fix

Add a `ModeAwaitingLLM` case immediately inside the existing ESC block at `tui.go:1252`. The branch should (a) call `m.inFlightCancel()` if non-nil, (b) transition to `ModeMenu`, and (c) call `m.menuSystem.Open()`. This mirrors what Ctrl+C does for cancellation, and what the existing ESC branch does for menu entry:

```go
if msg.Type == tea.KeyEsc {
    if m.mode == ModeOnPath || m.mode == ModeOffPath {
        m.mode = ModeMenu
        m.menuSystem.Open()
        return m, nil
    }
    // NEW: ESC during an in-flight turn cancels the turn and opens the menu.
    if m.mode == ModeAwaitingLLM {
        if m.inFlightCancel != nil {
            m.inFlightCancel()
        }
        m.mode = ModeMenu
        m.menuSystem.Open()
        return m, nil
    }
}
```

The comment on the existing guard ("It deliberately does not fire during ModeAwaitingLLM — Ctrl+C cancels the turn first") should be removed or updated, as this constraint is lifted by the fix.
